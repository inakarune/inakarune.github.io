알고리즘 소개

알고리즘의 개념

컴퓨터를 이용한 문제 해결 능력은 주어진 문제에 대한 알고리즘의 존재 유무가 결정
컴퓨터과학 : “알고리즘과 관련된 이슈를 다루는 학문”
알고리즘 : 주어진 문제를 해결하기 위한 명령어들을 단계젹으로 나열한 것
만족해야 할 조건 -> 입출력, 명확성, 유한성, 유효성

  • 알고리즘 생성 과정 : 설계 -> 기술(표현) -> 정확성 분석 -> 효율성 분석
    • 기술 방법 -> 일상적인 언어, 순서도, 의사 코드 등

기본 자료구조

좋은 프로그램은 주어진 문제에 적합한 자료구조와 효율적인 알고리즘을 사용한다.

  • 선형 자료구조(배열, 연결리스트, 스택, 큐) <-> 비선형 자료구조(트리, 그래프)
  • 배열 : 같은 자료형을 갖는 여러 원소를 하나의 변수이름으로 모아놓은 데이터의 집합
    • 인덱스를 통한 직접적인 원소 접근 -> 빠르고 동일한 접근 시간
    • 삽입/삭제 시 추가적인 자료의 이동에 따른 시간적인 오버헤드 발생
  • 연결 리스트 -> 데이터 필드와 링크 필드로 이루어진 노드라는 저장구조를 이용
    • 비교적 간단한 삽입과 삭제 과정
    • 순차적인 데이터 접근
  • 스택 : 한쪽 끝에서만 데이터의 삽입/삭제가 수행되는 선형 리스트
    • 후입선출(LIFO), push 연산, pop 연산, top
  • 큐 : 선형 리스트의 한쪽 끝에서는 데이터의 삽입만 수행되고, 다른 한쪽 끝에서는 삭제만 수행되는 자료구조
    • 선입선출(FIFO), front/head, rear/tail
  • 트리 : 하나 이상의 노드로 구성된 유한 집합 T
    • 조건1 T의 원소 가운데 단 하나의 루트 노드가 존재한다.
    • 조건2 루트 노드를 제외한 나머지 노드는 n개(n >= 0)의 서로 분리된 부분 집합 T1, T2 … Tn으로 나누어지며, 각 Ti는 트리가 된다. 이 때 각 Ti를 트리 T의 서브트리라고 한다.
    • 용어 : 노드의 차수, 트리의 차수, 리프 노드(단말 노드), 비단말 노드, 부모 노드, 자식 노드, 형제 노드, 후손(자손), 조상(선조), 높이(깊이), 숲
    • 이진 트리 : 각 노드의 차수가 2 이하인 순서 트리
      • 레벨i에서의 최대 노드의 개수 -> 2i
  • 종류 : 전 이진 트리, 포화 아진 트리, 완전 이진 트리, 균형 이진 트리 등
  • 그래프 G = (V, E) -> V : 정점의 집합, G : 간선의 집합
    • 간선의 방향성 유무에 따라 -> 무방향 그래프, 방향 그래프
    • 가중 그래프 -> 간선에 비용이나 시간과 같은 의미를 갖는 가중치를 부여
    • 용어 : 인접, 부수, 부분 그래프, 경로, 경로의 길이, 차수(진입 차수, 진출 차수), 단순 경로, 사이클, 루프, 연결, 강력 연결
    • 구현 방법 : 인접 행렬, 인접 리스트

알고리즘의 설계

주어진 문제, 속성, 조건 등이 매우 다양하므로 모든 문제에 일반적으로 적용할 수 있는 설계 기법은 존재하지 않음
대표적인 설계 기법 : 분할정복 방법, 동적 프로그래밍 방법, 욕심쟁이 방법
문제 기반의 알고리즘 -> 정렬 알고리즘, 탐색 알고리즘

알고리즘의 분석

정확성 분석 -> 유효한 입력에 대해 유한 시간 내에 정확한 결과를 생성하는지 판단
효율성 분석 -> 알고리즘 수행에 필요한 메모리의 양과 수행 시간을 측정

- 공간 복잡도 -> 메모리의 양을 측정
- 시간 복잡도
    - 알고리즘의 수행 시간 = 알고리즘에서 수행되는 단위 연산(명령)의 수행 횟수의 총합
    - 입력 크기(입력으로 제공되는 데이터의 크기)의 함수로 표현
    - 입력 데이터의 상태에 종속적 -> 최악의 수행 시간으로 표현

점근성능

입력 크기가 무한대로 커짐에 따라 결정되는 알고리즘의 성능

- 다항식으로 표현되는 수행 시간에 대해서 입력 크기가 커짐에 따라 차수가 낮은 항들과 상수항의 역할은 감소하므로, 결국 계수 없이 최고차항만을 이용해서 표시 -> 알고리즘의 정확한 수행 시간이 아니지만, 입력 크기가 증가함에 따라 알고리즘의 수행 시간이 어떤 추세로 증가하는지를 의미 -> 알고리즘의 우열을 따질 때 용이

표기법

- O 표기 -> 점근적

순환 알고리즘의 성능

  • 순환(재귀) -> 함수의 수행 과정에서 자기 자신의 함수를 다시 수행하는 형태
  • 분할정복 방법을 적용한 알고리즘들은 개념적으로 순환 알고리즘의 형태를 가짐 -> 수행 시간은 점화식으로 표현
