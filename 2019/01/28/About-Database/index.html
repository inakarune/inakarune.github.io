주요용어

데이터베이스 관리 시스템(DBMS), 스키마, 인스턴스, 데이터 모델, 트랜잭션, DBMS 3단계 구조

데이터베이스의 이해

데이터베이스 시스템의 개요

데이터베이스 : 데이터의 집합
데이터베이스 관리 시스템(DBMS: Database Management System) : 컴퓨터를 사용하여 데이터베이스를 관리할 수 있는 시스템.
DBMS : 한 조직의 연관된 데이터의 집합을 다수의 사용자가 공용으로 사용하기 위해 통합 저장하는 소프트웨어 패키지.

  • 목적 : 데이터베이스 내의 데이터를 편리하고 효과적으로 저장하고 조작.

데이터베이스 시스템(database system)

  • DBMS와 함께 사용자에게 서비스 형태로 제공되는 애플리케이션이 포함된 일체의 시스템
  • 대량의 정보를 관리하기 위한 목적으로 설계
  • 정보 저장을 위한 구조 정의와 정보 조작을 위한 메커니즘이 포함되어야 함.
  • 다수의 사용자가 데이터에 접근하기 때문에 오류 및 비허가된 접근의 시도에도 저장된 정보의 안정성을 확보할 수 있어야
  • 데이터가 여러 사용자에게 공유되는 환경에서 발생할 수 있는 비정상적(anomalous) 결과를 배제할 수 있어야

데이터베이스 관리 시스템의 목적

DBMS는 컴퓨터를 이용하여 상업적인 데이터를 관리하기 위한 목적으로 사용되기 시작

파일 처리 시스템(file processing system)

  • 파일을 사용하여 데이터를 관리하는 방식
  • 운영체제에 의해 지원
  • 운영체제는 여러 파일에 레코드(데이터 단위)를 기록, 레코드를 조작하기 위한 별도의 프로그램을 필요
  • 즉, 운영체제의 지원으로 여러 파일에 나누어 정보를 영구 저장하고 운영하는 시스템
  • 데이터의 종속, 데이터의 중복, 데이터의 무결성 훼손 및 동시 접근 이상과 같은 문제점 있음

데이터의 종속

  • 파일 처리 시스템에서는 파일들이 특정 프로그램에 종속됨
  • 데이터가 특정 프로그램에 종속될 경우, 데이터의 논리적인 구조나 위치의 변경이 애플리케이션 프로그램의 변경으로 이어지기 때문에 프로그램 유지보수에 많은 비용이 소요
  • 다른 프로그램과의 데이터 공유도 매우 어려워지게 됨

데이터 독립성(data independency)

  • 데이터와 애플리케이션 프로그램은 독립적으로 관리되어야 함.
  • 데이터를 효과적으로 활용하기 위해 반드시 필요한 전제조건
논리적 데이터 독립성(logical data independency)
물리적 데이터 독립성(physical data independency)

데이터의 중복

파일 처리 시스템에서 여러 파일에 중복적으로 저장될 수 있다
데이터 중복성 : 데이터가 여러 파일에 중복되어 저장되는 문제

  • 일관성(consistency) 문제: 하나의 사실을 나타내는 여러 개의 데이터가 존재할 때, 모든 데이터의 값이 논리적으로 같은 것.
  • 보안성(security) 문제 : 논리적으로 동등한 내용의 데이터에 대해서는 똑같은 수준의 보안이 유지되어야 한다.
  • 경제성(economics) 문제 : 데이터를 중복 저장하기 위해서는 추가적인 저장 공간이 필요하다. 또한 데이터 갱신 작업은 시스템의 갱신 비용이 높아진다.

DBMS는 데이터의 종속 및 중복 문제를 해결하기 위해 제안된 시스템

데이터의 무결성 훼손

데이터 무결성(data integrity) : 데이터베이스에서 관리되는 데이터의 정확성을 보장하는 것.
중복된 데이터가 서로 일치하지 않는 경우에도 발생.
중복이 발생하지 않는 경우에도 부정확한 데이터가 데이터베이스 안에 저장될 수 있다.
무결성 유지 : 데이터베이스 내에 훼손 문제를 방지하는 작업

  • 중앙통제에 의한 갱신 방법 : 검증 프로그램을 작성하여 모든 갱신 처리 작업 시 필수적으로 검증을 거치도록 통제한다.

동시 접근 이상

DBMS에서 동일한 데이터에 대한 수정 요구가 동시에 발생한 경우 비정상적인 데이터 수정이 일어날 수 있다.
동시성이 제어되지 않는 시스템에서는 예측할 수 없는 잔액이 나타날 수 있다.
예측 불가능한 값이 발생하는 문제를 막기 위해 시스템은 어떤 형태로든 데이터 접근에 대한 통제를 가해야 한다.
DBMS에서는 이를 해결하기 위해 트랜잭션이라는 단위의 개념을 사용함으로써 동시 접근(concurrent access) 기능을 제공한다.

데이터베이스 관리 시스템의 특징

DBMS에서는 데이터가 중복되지 않도록 관리하고 여러 사용자가 동시에 접근 가능

자기 기술성

데이터 자체뿐만 아니라 데이터에 대한 정의나 설명에 대한 부분까지 관리한다
시스템 카탈로그는 특정 데이터베이스에 저장된 파일의 구조를 파악하기 위한 데이터의 타입이나 포맷과 같은 정보를 담고 있다.
시스템 카탈로그에서 데이터베이스에 대한 정의를 참조하여 데이터베이스에 접근하므로 다양한 방법으로 데이터에 접근할 수 있다.

프로그램과 데이터의 독립성 및 추상화

파일 처리 시스템에서는 데이터 파일이 애플리케이션 프로그램 내에 종속되어 있다
DBMS는 데이터 파일의 구조가 프로그램으로부터 분리되어 시스템 카달로그에 저장된다.
프로그램-데이터 독립성(program-data independency)

다중 뷰

뷰(view) : 한 데이터 베이스에 대해 필요한 부분만 추출해서 볼 수 있도록 한다.
DBMS는 각각 사용자가 마치 독자적으로 데이터베이스에 접근하는 것처럼 보일 수 있도록 여러 개의 뷰를 제공한다.

다수 사용자 요청 처리

DBMS 는 여러 사용자가 동시에 데이터베이스에 접근할 수 있는 기능을 제공한다.
특히 다수 사용자용 DBMS는 단일 논리적인 작업을 수행하는 일련의 명령문의 집합인 트랜잭션(transaction)과 동시성 제어(concurrency control)라는 기능이 내장되어 있다.
데이터의 일관성을 보장할 수 있다
